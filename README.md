## Overview

This repository contains daily earthquake probability forecasts generated by the V200 Global model. Each day, the model produces probability estimates for seismic events across four magnitude tiers, covering the next 2-3 days.

The forecasts are designed to support:
- Emergency preparedness planning
- Insurance risk assessment
- Infrastructure monitoring decisions
- Research into seismic patterns

## Calibration Files

The `/calibration` folder contains static model configuration files committed once at deployment. These files define how predictions are generated and how alert thresholds are set.

### `v200_global_calibrated_thresholds.json`

Contains the probability thresholds used to generate alerts at each budget level for each tier:

```json

  "training_date": "2026-01-21",
  "calibration_period": {
    "start": "2018-01-01",
    "end": "2026-01-22"
  },
  "tiers": {
    "T1_micro": {
      "magnitude": 2.5,
      "window_days": 2,
      "thresholds": {
        "0.5": 0.892,
        "1.0": 0.847,
        "2.0": 0.782,
        "5.0": 0.654,
        "10.0": 0.521
      }
    },
    "T4_damage": {
      "magnitude": 6.0,
      "window_days": 3,
      "thresholds": {
        "0.5": 0.142,
        "1.0": 0.089,
        "2.0": 0.054,
        "5.0": 0.028,
        "10.0": 0.015
      }
    }
  }
}
```

**Key Fields:**
| Field | Description |
|-------|-------------|
| `model_version` | Identifier for the deployed model version |
| `calibration_period` | Historical period used to calibrate thresholds (8 years of recent data) |
| `thresholds` | Probability cutoffs for each alert budget level â€” when a zone's probability exceeds the threshold, an alert is triggered |
| `magnitude` | Minimum earthquake magnitude for this tier |
| `window_days` | Forecast horizon (how many days ahead the prediction covers) |

### `v200_global_active_zones.json`

Lists all 563 zones for which the model generates daily predictions:

```json
{
  "n_zones": 563,
  "zones": ["Z_-60_-75", "Z_-55_-70", "Z_35_-120", ...]
}
```

This file defines the complete set of zones included in the daily prediction output.

### `model_metadata.json`

General model information including version identifier, training completion date, and high-level configuration summary (without revealing proprietary model details).

## Understanding the Metrics

### Confusion Matrix Terms

| Term | Definition |
|------|------------|
| **Hit (True Positive)** | Alert was issued AND an earthquake occurred in the window |
| **Miss (False Negative)** | No alert was issued BUT an earthquake occurred |
| **False Alarm (False Positive)** | Alert was issued BUT no earthquake occurred |
| **True Negative** | No alert was issued AND no earthquake occurred |

### Performance Metrics Explained

| Metric | Formula | What It Means |
|--------|---------|---------------|
| **Precision** | Hits Ã· (Hits + False Alarms) | "When we alert, how often are we right?" Higher precision means fewer false alarms per alert issued. |
| **Recall** | Hits Ã· (Hits + Misses) | "Of all earthquakes that occurred, what fraction did we alert for?" Higher recall means we catch more events. |
| **F1 Score** | 2 Ã— (Precision Ã— Recall) Ã· (Precision + Recall) | Harmonic mean of precision and recall â€” balances both concerns. |
| **False Alarm Rate (FAR)** | False Alarms Ã· (False Alarms + True Negatives) | "What fraction of quiet days triggered alerts?" Lower is better. |
| **Lift** | Precision Ã· Base Rate | "How many times better than random guessing?" A lift of 8Ã— means our alerts are 8 times more likely to precede an earthquake than random chance. |
| **Base Rate** | Total Events Ã· Total Predictions | Natural frequency of earthquakes â€” what you'd achieve by random guessing. |
| **Alert Rate** | (Hits + False Alarms) Ã· Total Predictions | Fraction of zone-days that triggered alerts at this budget level. |

### Interpreting the Metrics

| Metric | Good Value | Interpretation |
|--------|------------|----------------|
| **Precision** | > 0.20 for T4 | 1 in 5 alerts results in an event (very good for rare events) |
| **Recall** | > 0.70 | Capturing 70%+ of all damaging earthquakes |
| **Lift** | > 5Ã— | 5Ã— better than random â€” strong predictive signal |
| **FAR** | < 0.02 | Less than 2% of quiet days trigger false alarms |

### Rolling Windows

Metrics are computed over three time horizons:

| Window | Purpose |
|--------|---------|
| **7-day** | Recent performance snapshot (more volatile, useful for detecting sudden changes) |
| **30-day** | Medium-term trend (recommended for routine monitoring) |
| **90-day** | Long-term baseline (most stable, best for overall model assessment) |

## Understanding Probability and Alerts

### Probability Values

Each zone receives a probability estimate (0.0 to 1.0) for each tier, representing the likelihood of at least one earthquake at or above that magnitude occurring within the forecast window.

### Alert Budgets

Alerts are triggered when a zone's probability exceeds a calibrated threshold. The budget levels represent different sensitivity settings:

| Budget | Meaning | Typical Use Case |
|--------|---------|------------------|
| **0.5%** | Highest threshold, fewest alerts (~4 alerts/day globally) | Critical infrastructure, nuclear facilities |
| **1.0%** | Very selective (~8 alerts/day) | High-consequence decisions |
| **2.0%** | Selective (~17 alerts/day) | Elevated preparedness |
| **5.0%** | Moderate (~42 alerts/day) | General monitoring |
| **10.0%** | Sensitive (~85 alerts/day) | Broad awareness, research |

**The budget percentage indicates approximately what fraction of zone-days will trigger alerts.** At 1% budget, roughly 1% of all daily zone predictions will be flagged.

## Verification & Integrity

### Immutable Timestamps

Each prediction file is committed to GitHub **before** the forecast window begins. The Git commit timestamp provides cryptographic proof that predictions were made in advance, enabling independent verification by any third party.

### Data Integrity

| Field | Purpose |
|-------|---------|
| `data_hash` | SHA-256 hash of prediction content for tamper detection |
| Git commit hash | Immutable record of exactly when predictions were published |

Third parties can clone this repository and verify that predictions were committed before the earthquakes they predicted.
